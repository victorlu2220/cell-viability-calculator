Cell Viability Calculator

















Code Logic

The code first crops the uploaded image if it sees apparent grey or sharp green area. Then it recognizes cell-free regions (white) so that they don't interfere with the subsequent calculation; Finally, it runs cell viability calculation(dead_cell_area/overall_colored_area) based on the remaining part of the uploaded image.

Variable Definitions

  • Blue Min for Cropping: threshold for a row to be identified as a cell-free, vacant region. If the average of the blue values of the row is below this threshold, the row will be used as the starting row for our cropped image

  • Red Min for Cropping: threshold for a row to be identified as a cell-free dentin region. If the average of the red values of the row is below this threshold, the row will be used as the final row for our cropped image

  • Blue Min for Calculation: it checks if a point in space is grey enough, if it is, then it will be identified as cell-free region and won't involve in final calculation.

  • Red Min for Calculation: it checks if a point in space is red enough, if it is, then it will be identified as dead cell region.

    • Suggestions

      if excessive regions are being cropped on top, that means the first row is too "late". You should raise the "Blue Min for Cropping".
      if excessive regions are being cropped on bottom, that means the last row is too "early". You should lower the "Blue Min for Cropping".
      If you see excessive regions are whitened, then you should raise the Blue Min.
      Likewise, if you see excessive regions are reddened, then you should raise the Red Min.
